    Where this goes next

    Built as a starting point, not the finished product — here's the plan if this keeps growing inside E-Cell.

    1. 01

      Real-time scoring sync

      Push score updates over WebSockets, so mentors watching a live pitch session see results the instant a founder submits — not on refresh.
    2. 02

      Mentor analytics dashboard

      An aggregate view for E-Cell: average score per Launchpad cohort, the most common weak category across all founders, trends across the program's weeks.
    3. 03

      Event-day integration

      A kiosk / QR-code mode for Launchpad pitch day — founders score their pitch on a booth screen, straight into the same live leaderboard.
    4. 04

      Mentor annotations

      Let a mentor leave a short human note next to any AI score, so founders get both the AI read and real feedback in one place.

    Why I built this: Launchpad founders get one shot at a first impression with mentors — I wanted to build something that gives them a second pair of eyes before that moment, and prove I can ship a full working product: AI feature, backend, database, and CI, not just a static page.

    How it's built: the AI read comes from a small sentiment model (DistilBERT, via transformers.js) running locally via WebAssembly, combined with rule-based structural checks written from scratch. Scored pitches are saved through a serverless API (/api/evaluations) into a Postgres database on Supabase. Every push to main runs a CI check on GitHub Actions before Vercel deploys it live.
